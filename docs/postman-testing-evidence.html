Postman Testing Evidence

This document organizes the screenshots required for API testing. It also includes the SQL snapshot queries that should be run before and after each test to verify database changes in PostgreSQL.

How to Use This Evidence Sheet

  • Take one screenshot of the Postman request and response for each endpoint test.
  • Take one screenshot of the relevant database state before the request.
  • Take one screenshot of the relevant database state after the request.
  • Place the screenshots in the marked areas below before final submission.

Reference SQL File

The reusable snapshot queries are stored in database/verification_queries.sql. 
SELECT user_id, name, email, created_at
FROM users
WHERE email = 'amira.reader@example.com';

SELECT c.cart_id, u.email, ci.cart_item_id, b.title, ci.quantity, b.price
FROM cart c
JOIN users u ON u.user_id = c.user_id
LEFT JOIN cart_items ci ON ci.cart_id = c.cart_id
LEFT JOIN books b ON b.book_id = ci.book_id
WHERE u.email = 'demo@pageturn.com'
ORDER BY ci.cart_item_id;
