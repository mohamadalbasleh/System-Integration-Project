API Documentation Report

System: Online Bookstore

Purpose: This report documents the REST API contract required by the approved ER diagram and the Phase 1 frontend. The documented version is the corrected target contract, not the incomplete zipped backend as received.

System Overview

Base URLhttp://localhost:5000/api
DatabasePostgreSQL
AuthenticationJWT Bearer token for protected routes
Main EntitiesUsers, Categories, Books, Cart, CartItems, Orders, OrderItems
Primary ClientPhase 1 frontend in the bookstore/ folder

Authentication Rules

  • Public endpoints: register, login, categories, book listing, book detail.
  • Protected endpoints: cart and order endpoints.
  • Protected requests must send Authorization: Bearer <jwt_token>.

Endpoints

Notes on Database Alignment

  • The physical schema uses the ER diagram table names and key names such as user_id, cart_id, and order_id.
  • The API may expose these as shorter JSON keys such as id where needed by the frontend.
  • Money fields use PostgreSQL NUMERIC(10,2), not floating point types.
